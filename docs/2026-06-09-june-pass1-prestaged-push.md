## Why I can't push these myself, and the future fix

The write gap is real and worth fixing deliberately:

- **Google Sheet tracker:** my Drive MCP has read/search/copy/create-file tools but **no
  edit-cells/append-rows capability** (verified). I cannot write into the existing sheet.
- **Rundown Creator:** the app only **reads** from RDC (`rc-import` pulls RDC → Supabase).
  There is no write-to-RDC code, and no documented RDC write action; I won't guess at an
  undocumented production write.
- **Supabase (`graphics`/`scripts`):** per ADR-0012 this is the *intended* source of truth,
  and I can write there via the Supabase tooling — but the dashboard graphics page that
  would surface it isn't built, and the team works from the Sheet today.

**Recommendation (high confidence):** the durable fix is to stop pushing to the Sheet/RDC
at all and finish the **dashboard graphics page on Supabase `graphics`** (the ADR-0012
plan) — then "push" is a real, gated dashboard action I can drive. **Bridge until then:**
add a write path to the live tracker — either a **Google Sheets MCP with write scope** or a
tiny **Apps Script web-app endpoint** on the June sheet that accepts rows. That single
addition would let me (and later the dashboard) write the tracker directly, ending the
copy-paste. RDC write-back is optional and lowest priority — confirm whether its API even
supports a write action before investing.
