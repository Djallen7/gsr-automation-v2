Part 1 // The Pipeline ProcessWhat we know about the show, end to end.

The show

Genesis Science Report (GSR) is a Christian creation-science TV show for David Rives Ministries. About 58 minutes, weekly, currently Season 3. David Rives is the on-screen talent and ministry director.

Batches of 5. The team runs roughly 5 episodes in pre-production and 5 in post at the same time (ten in flight). TV airs Tuesday 8 PM CST; YouTube publishes Monday 4 PM ET. Five Season-3 episodes were filmed May 28-29; all 48 episodes are already in the database.

The two governing rules

The David Rule. Before any action, ask "if this goes wrong, does it land on David to fix?" If yes, redesign until the answer is no. In practice: automation may write the database and call cloud APIs, but it never touches the live broadcast chain. A human always loads the live gear. This is the one hard line.

The maturity dial. Every generative task moves through three stages, flipped per feature in app_config as trust grows: Manual (Daniel does it, the system stores it), Prompt handoff (the system hands a context-loaded prompt, Daniel pastes it into Claude and pastes the result back through the same dry-run + Type YES gate), then Auto (the system makes the call itself, same gate). Automation is the end goal from day one; the dial just controls how fast each feature gets there. Even on an aggressive pace, high-stakes steps ship manual-trigger first.

Off-limits infrastructure (non-negotiable)

  • ProPresenter production machine (GSN-PropRes, Tailscale 100.98.215.7) -- off-limits to automation until David approves a tested path.
  • ATEM switcher and Bitfocus Companion -- production hardware, never driven by the system.
  • QNAP -- read-only SMB only. No admin, no writes, no file-watchers (QNAP3 10.2.2.3, QNAP5 10.2.2.5).
  • Tailscale and direct server tools -- permanently barred after the 2026-05-20 security incident. All automation goes through cloud APIs or read-only SMB.
  • Notion -- wiki-only after ADR-0012.

Things that look proposed-vs-applied, stale, or worth a second look

These are real and openly flagged in the source. They are not bugs in this document; they are items where the plan and the live system have not yet been reconciled. Listing them so nothing slips.

1. graphics -> lower_thirds rename (proposed, NOT applied). The table that holds lower thirds is still named graphics in the live database. The course and canon both call for renaming it to lower_thirds (keeping production_graphics as the Graphics Tracker). Until the prereq migration runs, every query still uses graphics and the only thing named lower-thirds is the storage bucket. m1 and m2 depend on this rename.

2. premade_library / Dreamstime enum (proposed, NOT applied). The course prereq says to add Dreamstime to "a new b-roll source enum (premade_library; it does not exist yet)." Canon section 8, however, records that a premade_library table and a b-roll source enum already exist (migration 20260528005100), with values envato, storyblocks, nasa_svs, creation_com, own_production, other. So the table likely exists and what is actually missing is just the Dreamstime value. Worth confirming whether the build adds a value to the existing enum or creates a new one; the two source docs phrase it differently.

3. production_graphics.status CHECK order (wrong in the DB). The live CHECK lists the statuses in the wrong order (Not Started, Created, In Progress, Loaded In) versus the canon order (Not Started -> In Progress -> Created -> Loaded In). Prereq 3 fixes this and restores 'In Progress' in PROMPT_LIBRARY. Cosmetic for the four values, but the order matters for the UI.

4. Book Cover graphic type (used in the sheet, missing from the DB). Book Cover appears in the live Google Sheet but is missing from the production_graphics.graphic_type CHECK. Prereq 2 adds it.

5. GSN Roku channel-type assumption (unverified). The Roku Direct Publisher JSON feed for GSN only works if the channel is feed-driven (Direct Publisher) rather than a custom SDK channel. This is assumed, not confirmed. It is a Setup item still flagged "verify."

6. RLN -20 LKFS requirement (survives only in a superseded migration). The -20 LKFS audio normalization for Real Life Network currently lives only in superseded migration ..002000, with nothing enforcing it. The course says to recapture it when RLN delivery is built.

7. CTN and WWN enum values (named, not yet written). Prereq 6 adds creation_tv_network and wwn to the distributions.platform enum. These are real low-cadence targets; the migration is named but not yet written.

8. Naming: Myriam vs Miryam. The role notes (2026-06-07) spell her name Myriam; earlier canon entries spell it Miryam. Same person (metadata, thumbnails, uploads, mark-aired, Daniel's successor). Worth settling one spelling.
