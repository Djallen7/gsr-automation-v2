# GSR Dashboard UI Design Package

A cohesive UI design package for the Genesis Science Report production dashboard, produced 2026-06-07 by a 12-agent design team (brand, density, data-viz, information architecture, components, mobile, lower-thirds, graphics tracker, distribution, motion, sources, accessibility).

## What is here

| File | What it is |
|---|---|
| **GSR-UI-DESIGN-STRATEGY.md** | The main report: design advice, the GSR design language (cosmos navy + broadcast gold, OKLCH tokens, fonts, motifs), the principles, the per-role information architecture, the high-value screens, motion, accessibility, and a recommended build order. Start here. |
| **COMPONENT-CATALOG.md** | A catalog of compact, purpose-built UI elements mapped to the real pipeline, each buildable on shadcn/Tailwind v4, plus the libraries worth adopting and the shadcn primitives still to add. |

| **gsr-dashboard-mockup.html** | An interactive, mobile-friendly, brand-styled mockup. Open it in any browser. Toggle the theme (moon button), switch between the Owner Dashboard, Graphics Master, Lower-thirds Review, Distribution, and Mobile views, and resize the window narrow to watch it reflow to the phone layout with a bottom tab bar. |

## The short version

A stunning, non-generic dashboard comes from three moves: a distinct GSR brand skin (deep-space navy lit by broadcast gold, a faint starfield, a lower-third bar motif), ruthless reduction (one bold number and one obvious action per screen), and a small set of clever purpose-built components (a pipeline rail, a one-strip show-status bar, a character-count meter that doubles as a quality gauge). Beauty and function are not a trade. The mockup shows both at once.

These documents are a strategy, not a started build. Per the project Anti-Churn Rule, the recommended deliverables and ship order live in section 8 of the strategy doc.
