# GSR Dashboard UI Design Package (v2)

A fresh, ground-up design effort for the Genesis Science Report production dashboard, 2026-06-07. Produced by **5 teams of 6 agents (30 total)**: Pipeline Discovery, Brand (violet/black), Dashboard Inspiration, Custom-Element Invention, and Synthesis/Validation. Grounded in the repo and real web research.

**This package supersedes `docs/ui-design/`** (the earlier navy/gold package), per Daniel's direction on 2026-06-07.

## The new ground rules (from Daniel)
- The operation runs on a **monthly cycle**, not weekly: **5 episodes in pre-production at once**, moving through the pipeline in parallel. The dashboard's spine is this monthly batch of 5.
- North star: open it any day and **instantly know where you are in the cycle**. Intuitive, efficient, sleek, not overwhelming.
- Brand colors are **violet and black**, and it must **not read feminine**.

## The short version
A stunning, non-generic result comes from three things together: a **distinct violet-black skin** with Linear-grade restraint (violet rationed to interaction, on near-black), a **monthly-cycle spine** that organizes everything around your batch of 5, and a **small set of custom components** that each carry a lot in a tiny footprint. The recommended direction is the **Command Deck** (a calm status-page for your month) with a static cycle ribbon for rhythm; the Pipeline Rail and Cycle Dial are offered as alternates in the mockup.

These documents are a strategy, not a started build. The phased, named deliverables are in section 7 of the strategy doc.
